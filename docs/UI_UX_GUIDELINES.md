# UI/UX Guidelines

## Premium Dashboard Design Principles
- Make the dashboard canvas the primary workspace.
- Use clean white dashboard surfaces with dark navigation and mint/teal highlights.
- Keep KPI cards compact, clickable, and explanation-ready.
- Use strong spacing and readable chart labels.
- Prefer actual uploaded-data visuals over placeholder charts.

## Layout Standards
- The dashboard canvas should use most of the screen.
- Avoid horizontal page scrolling.
- Keep setup tables and package comparison tables scrollable inside their own containers.
- Use responsive grids that collapse before content overflows.
- Keep long survey labels truncated in controls, with full labels available in details or tooltips.

## Interactive Preview Toolbar
- Include close, fit to screen, zoom in, zoom out, export menu, and upgrade status.
- Step 6 should open reliably after preview generation.
- The full-screen workspace should preserve locked preview watermarks.

## Package Card Standards
- Cards should explain outcomes, not internal implementation details.
- Keep pricing visible but subtle.
- Avoid repeated price language inside analytics sections.
- Selected cards should feel active and valuable.

## Output Level Selector Standards
- Use Essential, Professional, Premium, and Executive language.
- Avoid old tier language.
- Explain output polish and deliverables clearly.

## Chart Card Standards
- Chart titles should be plain language.
- Date fields should appear as grouped trends.
- Categorical charts should show top values only.
- Missing values should use horizontal bars and heatmaps.
- Quality should show a score plus component breakdown.

## Locked Preview Behavior
- Locked previews must be useful but protected.
- Use limited rows, limited charts, blurred or muted premium areas, and watermarks.
- Disable full export, full copy, and full download until checkout unlocks the package.

## Watermark Behavior
- Use "ProgramMetrics Preview" on locked charts, tables, dashboards, and reports.
- Watermarks should protect the output without making the preview unreadable.

## Copywriting Standards
- Use customer-facing language.
- Use "package" and "output level" instead of "tier".
- Do not put pricing inside analytics summaries.
- Do not claim conclusions the dataset cannot support.
- Prefer "Upgrade to export" over pressure-heavy copy.

## Hard Rules
- No "tier" language in new UI copy.
- No pricing inside analytics summaries.
- No placeholder charts.
- No oversized preview headers.
- Dashboard canvas should use most of screen.
- Locked previews are useful but protected.
