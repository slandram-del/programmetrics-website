Refund Policy

ProgramMetrics provides digital products and customized services. Because these products are digital and, in many cases, generated specifically for each customer, all sales are generally final.

Tier 1 - Dashboard Generator

Refunds may be requested within 7 days of purchase only if a verified technical issue prevents the dashboard from being successfully generated or downloaded.

A verified technical issue is a problem that ProgramMetrics can reproduce or confirm, including:

  • A platform or server error that prevents dashboard generation.
  • A software bug that prevents a dashboard from being created from a supported file type containing valid data.
  • A successful payment where the purchased service is not delivered.

Once a dashboard has been successfully generated and made available for download, the order is considered complete and is not eligible for a refund.

Refunds are not available for:

  • Incorrect, incomplete, or corrupted uploaded data.
  • Unsupported file formats.
  • User error or incorrect configuration.
  • Changes of mind.
  • Dissatisfaction with dashboard appearance or layout.
  • Errors originating from customer-provided data.

Tier 2 - Dashboard Customization

Tier 2 customizes an existing ProgramMetrics dashboard using the customer's submitted configuration options.

Because each dashboard is customized specifically for the customer, all Tier 2 sales are final once customization has begun.

If ProgramMetrics incorrectly applies a supported customization, we will correct the issue at no additional charge. Corrections are the customer's exclusive remedy and do not entitle the customer to a refund.

If ProgramMetrics cannot deliver the purchased customization because of a verified technical issue that cannot be resolved, ProgramMetrics may, at its sole discretion, issue a full or partial refund.

Tier 3 - Custom Applications & Enterprise Solutions

Tier 3 projects are governed by a separate proposal, quote, Statement of Work (SOW), or service agreement. Payment schedules, acceptance criteria, change requests, support, and any refund provisions are defined in those project documents.
