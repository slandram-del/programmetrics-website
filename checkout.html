Recommended payment setup

Use Stripe as the payment record, not a customer-file database.

  • Create one checkout price for each package and output level.
  • Create upgrade checkout links for package and output-level price differences customers can buy later.
  • After successful checkout, redirect back to Studio with a signed access token.
  • Keep uploaded customer files temporary: process, download, discard.

For launch, ProgramMetrics can use Stripe to remember payment/customer history while Studio avoids long-term customer file storage.

Codex side: paste sandbox Payment Link URLs into stripe-links.js. The checkout button will automatically use the correct full-purchase or upgrade link.
